## Context

右侧设置面板此前为了排障加入了分层调试色和内容高度收缩策略，导致面板视觉高度与预期不一致。用户现要求明确且可量化：面板全高，且上/右/下边距均为 16pt。

## Goals / Non-Goals

**Goals:**
- 右侧面板在 trailing 方向严格满足 16pt 上/右/下边距。
- 设置页内容容器恢复全高占位。
- 清理调试上色代码。

**Non-Goals:**
- 不修改倍速面板底部大间距。
- 不调整其他侧栏（字幕/增强/音轨/视频轨）业务交互。

## Decisions

- 决策 1：在 `SiderContanierView` 采用方向区分外边距：trailing=16，bottom 维持原值。
  - 原因：精准满足右侧面板要求，同时隔离对倍速面板的影响。
- 决策 2：恢复 `SiderSettingView` 内容最大高度填充，移除 `PlaySettingPage` 的 macOS 垂直收缩。
  - 原因：用户要求“全高”，内容层应跟随容器高度。
- 决策 3：移除调试颜色参数与背景修饰。
  - 原因：该代码仅用于排障，不应进入最终样式。

## Risks / Trade-offs

- [Risk] 右侧面板变高后底部空白会增加。→ Mitigation：这与“全高+16pt边距”需求一致。
- [Risk] 方向分支边距差异可能影响未来统一样式。→ Mitigation：变量命名明确，后续可集中调整。
