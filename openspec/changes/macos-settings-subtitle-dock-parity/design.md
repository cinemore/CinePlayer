## Context

当前播放器已完成一轮业务按钮对齐，但在 macOS 播放设置、字幕面板样式和文件打开链路上仍与 Cinemore 参考实现存在差异。差异直接影响可用性：设置入口出现非预期项、字幕调整不可直达一致路径、以及 Dock 拖入文件触发重复窗口。

## Goals / Non-Goals

**Goals:**
- 让 macOS 播放设置页面与 Cinemore 的平台差异保持一致：仅保留左右键步进秒数选择器，移除长按倍速与无关分组。
- 让 iOS/macOS 字幕面板的分段结构、行样式与子页交互一致，并保留可操作的字幕调整功能。
- 让 macOS 文件打开事件在拖入 Dock 图标时只创建/激活一个播放器窗口。

**Non-Goals:**
- 不引入“换源”“剧集列表”“外部字幕从文件导入”等被明确排除能力。
- 不改动解码器能力、媒体解析链路和播放器核心播放状态机。

## Decisions

1. 平台化设置分组严格对齐 Cinemore
- macOS 设置页只呈现“左键后退秒数”“右键前进秒数”两个 menu picker。
- iOS 继续保留其移动端差异项；macOS 不渗透移动端交互。
- 备选方案：使用统一分组+条件隐藏字段。未选原因：会留下布局空洞和文案偏差，难以确保视觉完全一致。

2. 字幕面板采用与 Cinemore 同构的分段和样式
- 保持“内封字幕 / 外部字幕 / 字幕调整”分段及对应子视图入口样式。
- 字幕调整提供可直接生效的偏移调节，面板中不出现占位提示。
- 备选方案：保留现有简化字幕页。未选原因：已被用户验证为不一致。

3. macOS 文件打开单路径去重
- 将 macOS 文件打开统一收敛到 AppDelegate 通知链，避免同一 URL 同时被 `onOpenURL` 和 `application(_:open:)` 重复消费。
- 通过平台条件编译或去重哨兵确保同一事件只触发一次窗口打开。
- 备选方案：保持双通道并依赖运行时节流。未选原因：时序不稳定，仍可能重复开窗。

## Risks / Trade-offs

- [风险] UI 样式对齐后可能影响既有间距/可访问性细节 → [缓解] 仅替换目标分组与行样式，保留系统字体缩放和可点击区域。
- [风险] 去重逻辑误伤合法二次打开 → [缓解] 去重范围限定在同一事件周期与同一 URL。
- [风险] 字幕调整与播放器时钟绑定可能出现边界值问题 → [缓解] 沿用现有偏移写入路径，仅修正入口与控件呈现。
